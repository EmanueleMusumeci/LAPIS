\begin{abstract}
We demonstrate \textbf{LAPIS} (\emph{Language-Adaptive PDDL Iterative Synthesis}), an interactive system for end-to-end natural language to plan synthesis. Given an NL task description, LAPIS synthesizes both PDDL domain and problem files, invokes FastDownward for planning, and iteratively refines errors using VAL feedback. Our controlled evaluation across seven IPC domains reveals a counterintuitive finding: \emph{LLM-generated domains often yield higher success rates than human-authored ground truth} (e.g., Floortile: 94\% vs.\ 45\%), because self-consistent domain-problem pairs avoid coupling errors that arise when forcing an LLM to adapt to external PDDL conventions. The Streamlit-based demo enables users without PDDL expertise to input NL tasks and receive executable plans, visualizing each pipeline stage including domain generation, validation logs, and plan traces.

\section{Introduction}

Translating natural language task descriptions into executable plans remains a bottleneck for deploying AI planning in real-world applications. While recent work has shown that large language models can generate PDDL~\citep{llmpp2023, nl2plan2025}, single-shot synthesis frequently fails due to predicate hallucination, type mismatches, and domain-problem coupling errors that have no clear recovery path.

LAPIS addresses these challenges through schema injection and VAL-guided iterative refinement, building on the ContextMatters framework~\citep{contextmatters2025}. Rather than requiring few-shot examples or human-authored domain templates, LAPIS extracts the schema from its own generated domain and uses it to constrain problem synthesis, creating internally consistent PDDL pairs. When validation fails, the system prompts the LLM with error diagnostics and iterates until success or timeout.

Our evaluation demonstrates that this self-consistent synthesis approach handles diverse IPC domains without domain-specific tuning, recovers from approximately 30\% of initial synthesis failures through refinement, and scales to problems requiring 80+ step plans. The interactive demo requires only an API key---no PDDL expertise needed.

\section{System Architecture}

LAPIS translates NL task descriptions into executable plans through four stages (Fig.~\ref{fig:pipeline}). The system handles errors through iterative refinement, enabling recovery from synthesis failures that would be terminal in single-shot approaches. The same pipeline generalizes across IPC domains without modification, and problems with plan lengths exceeding 80 steps are handled within the standard timeout.

\paragraph{Domain Synthesis.}
Given an NL environment description, the LLM generates a PDDL domain encoding action schemas and predicates. An optional chain-of-thought \emph{adequacy check} verifies that the domain covers task requirements before proceeding---catching missing predicates (e.g., \texttt{shaker-clean} in Barman) that would otherwise waste refinement iterations.

\paragraph{Problem Synthesis with Schema Injection.}
LAPIS extracts exact type definitions, predicate signatures, and constants from the domain and injects them into the problem generation prompt. This \emph{schema injection} prevents predicate hallucination and type mismatches---the dominant failure mode in naive synthesis.

\paragraph{VAL-Guided Refinement.}
If FastDownward fails or VAL rejects the plan, LAPIS enters a refinement loop: capture errors, prompt the LLM with diagnostics, receive corrections, and retry (up to 3 iterations). A constraint locks the initial state, preventing the LLM from ``solving'' errors by hallucinating an easier world. The loop employs adaptive temperature, error memory, and hallucination guards to improve refinement convergence.

\paragraph{Semantic Verification.}
Beyond syntactic VAL checks, LAPIS performs structural analysis to catch guaranteed-unsolvable problems: unreachable goal predicates, dead-end actions, and unfixable preconditions (static predicates never established). This addresses the semantic gap that Planetarium~\citep{planetarium2025} quantifies. Domains passing semantic checks achieved 100\% success (blocksworld, grippers) versus 50--65\% for those with structural warnings.

\paragraph{Simulator-Grounded Refinement.}
An optional mode validates plans against a ground-truth simulator. After VAL passes, the plan executes step-by-step; if an action fails or the goal is not achieved, the execution trace is analyzed by the LLM to produce targeted refinement suggestions. This reveals only sensor-observable state---never GT structure---providing fair evaluation of semantic feedback value.

\paragraph{Results.}
Table~\ref{tab:results} reveals a counterintuitive finding: LAPIS with \emph{generated} domains (Synth) often outperforms LAPIS with ground-truth domains (GT). On Floortile, synthesis achieves 94\% versus only 45\% with GT; on Tyreworld, 75\% versus 0\%. This occurs because LLM-generated domain-problem pairs share consistent naming conventions and structural assumptions, while forcing the LLM to adapt to human-authored PDDL introduces coupling errors.

The LLM+P baseline achieves near-perfect scores (95--100\%) because it receives the exact GT domain---a significant advantage. Even so, LAPIS/Synth matches or exceeds this ceiling on 3/5 complete domains (Storage, Termes, Grippers), demonstrating that full synthesis is viable.

\section{Demo Description}

The LAPIS demo (Fig.~\ref{fig:demo}) provides an interactive Streamlit dashboard where users can:
\begin{itemize}[nosep,leftmargin=*]
  \item Input natural language task descriptions
  \item Select experimental conditions (GT domain, synthesis, adequacy check)
  \item View generated domain and problem PDDL side-by-side
  \item Inspect VAL validation logs and refinement iterations
  \item Step through the generated plan with state visualization
\end{itemize}

The system requires only an API key---no PDDL expertise or domain-specific configuration. The 10-minute video demonstrates the full pipeline on Blocksworld and Floortile, showing both successful synthesis and error recovery through refinement.

\section{Conclusion}

LAPIS demonstrates that end-to-end NL-to-plan synthesis is practical for IPC-class problems. The key insight---that self-consistent LLM-generated domains can outperform human-authored ground truth---suggests that domain-problem coupling deserves more attention in future NL-to-PDDL systems.

\paragraph{Limitations.}
The +Sim variant requires a resettable GT environment---valid for benchmarks and simulators, but not real-world robotics where actions are irreversible. A robot cannot ``undo'' a failed manipulation; the iterative refinement loop assumes unlimited retries at zero cost. We interpret +Sim results as an \emph{upper bound} on what execution feedback can achieve; extending LAPIS to irreversible scenarios---where plans must succeed on first attempt---is a key direction for future work.
