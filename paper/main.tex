\begin{abstract}
Established approaches exploiting commonsense in planning. We demonstrate \textbf{LAPIS} (\emph{Language-Adaptive PDDL Iterative Synthesis}), a system for end-to-end NL planning without manual PDDL. Given an NL task, LAPIS synthesises both PDDL domain and problem files, invokes a classical planner, and refines them via VAL-guided feedback. We contribute: (i)~\emph{dynamic schema injection}, constraining problem synthesis; and (ii)~\emph{CoT adequacy checks}, verifying domain expressivity. LAPIS outperforms single-shot baselines on seven IPC domains, reaching 100\% success on BlocksWorld.
\textbf{Video:} \url{https://example.com/lapis-demo-video}
\end{abstract}

\section{Introduction}
Grounding natural language (NL) tasks into symbolic PDDL is a bottleneck. LAPIS (extending ContextMatters~\citep{contextmatters2025}) addresses this by interleaving LLM-based PDDL synthesis with VAL-guided refinement and adequacy checking. We evaluate LAPIS on LLM+P benchmark domains~\citep{llmpp2023}. We focus on recovering from grounding errors that cause single-shot planners to fail, using an iterative loop combining classical planners and symbolic validators.

\section{Related Work}
Embodied agents increasingly rely on LLM-based task planning. Purely semantic planners like SayCan~\citep{ahn2022saycan} interpret intent well but frequently generate unfeasible plans due to a lack of formal domain grounding. To address this, hybrid architectures like LLM+P~\citep{llmpp2023} and DELTA~\citep{delta2024} delegate plan generation to classical solvers by translating natural language (NL) into symbolic PDDL. We compare directly against LLM+P as it represents the current state-of-the-art for single-shot PDDL translation. However, these systems often fail on complex tasks—such as those in the LexiCon benchmark~\citep{lexicon2025}—because they lack the ability to iteratively recover from translation or grounding errors. LAPIS improves upon these approaches by building on the ContextMatters framework~\citep{contextmatters2025} to introduce cross-modal adequacy checks and iterative refinement loops that verify domain expressivity before classical search begins.

\section{Methodology and Evaluation}

LAPIS translates natural language into executable plans through four stages (Fig.~\ref{fig:pipeline}). Given an NL task description, the system first performs \textbf{domain synthesis}: an LLM generates a PDDL domain $\Sigma_0$ encoding action schemas and type hierarchies. VAL~\citep{val2004} validates syntax; parse errors trigger re-prompting. Next, an optional \textbf{adequacy check} uses chain-of-thought (CoT) prompting to verify that $\Sigma_0$ can represent all observed world facts---if a predicate is missing (e.g., \texttt{shaker-empty} in barman), the domain is amended before proceeding. In \textbf{problem synthesis}, the LLM generates a problem instance $\mathcal{P}^i_k$ using \emph{dynamic schema injection}: type definitions and predicate signatures extracted from $\Sigma_0$ are injected into the prompt, preventing hallucinated predicates or mistyped arguments. Finally, FastDownward computes a plan $\pi$; if planning or VAL validation fails, the \textbf{refinement loop} $\Gamma_{\text{refine}}$ re-prompts the LLM with feedback $H_{fb}$ (validation errors or planner output) to correct $\mathcal{P}^i_k$, iterating up to three times.

\paragraph{Experimental Setup.}
We evaluate on two benchmarks. The \emph{LLM+P benchmark}~\citep{llmpp2023} provides 20 NL problem instances across seven IPC domains (blocksworld, barman, floortile, grippers, storage, termes, tyreworld). The \emph{Lexicon benchmark}~\citep{lexicon2025} offers larger-scale evaluation with 150+ problems per domain (blocksworld, logistics, sokoban). All experiments use Claude~Sonnet~4.6 for the LLM+P benchmark and GPT-4o for Lexicon; we selected GPT-4o for Lexicon as it provided the best balance between logical reasoning and PDDL formatting consistency.

\paragraph{Results.}
Table~\ref{tab:ablation} compares ablation conditions on LLM+P. With a ground-truth domain (LAPIS/GT), schema injection and refinement achieve 100\% across all tested domains. When the domain is also generated from NL (LAPIS/full), blocksworld reaches 85\%; adding adequacy checks recovers the remaining 15\%. On barman---the most challenging domain due to complex multi-step cocktail recipes---LLM+P fails entirely (0\%) even with the GT domain, while LAPIS/full achieves 5\% by detecting missing predicates via CoT checks. Table~\ref{tab:lexicon} compares LAPIS against frontier LLMs generating plans directly (without PDDL). High-capacity models (o3, Gemini~2.5) excel on simple domains but degrade on logistics and sokoban; LAPIS remains competitive, particularly on sokoban where autoregressive planning struggles with spatial constraints.

\section{Conclusion}

The experiments reveal that iterative synthesis recovers most grounding failures of single-shot LLMs: schema injection eliminates predicate hallucination, while VAL-guided refinement corrects residual encoding errors. Decoupling semantic grounding from combinatorial search also enables LAPIS to solve problems with long horizons ($>$80 steps) that pure autoregressive architectures cannot plan through. At the same time, LLM-generated domains are often suboptimal for search---barman timeouts indicate that structurally valid domains may still induce combinatorial explosion. Future work will explore automated domain optimization and integration of external knowledge graphs (via OntoBDI) for open-world grounding. The demo itself highlights near-zero modeling effort: attendees interact with a Streamlit dashboard where NL input yields executable plans, bypassing the traditional ``expert-in-the-loop'' bottleneck.
